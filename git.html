Git

  • ls - расшифровывается как "List files in the directory", что переводится как "перечислить все файлы в папке" или "вывести на экран список файлов в папке".
  • cd - расшифровывается как "Change directory", что переводится как "сменить папку" или "перейти в другую папку".
  • pwd - расшифровывается как "Present working directory", что переводится как "текущая рабочая директория". То есть команда pwd показывает, в какой сейчас папке мы находимся.
  • mkdir - расшифровывается как "Make directory", что переводится как "создать папку".
  • mv - расшифровывается как "Move", что переводится как "переместить", "передвинуть". С помощью нее можно переместить, переименовать или удалить файл
  • Git branch — отображает список локальных веток
  • Git branch branch_name — создаёт новую ветку
  • Git checkout branch_name — переключается на указанную ветку
  • Checkout -b branch_name — создаёт и сразу переключается на новую ветку.
  • Git branch -d branch_name — удаляет ветку после её слияния.
  • Git branch -D branch_name — принудительно удаляет ветку, даже если она не слита.
  • Git merge branch_name — сливает указанную ветку в текущую.
  • Git pull — получает изменения из удалённого репозитория и автоматически сливает их с локальной веткой.
  • Git push — передаёт сделанные локально изменения в связанный удалённый репозиторий для их сохранения и совместной работы.
  • Git push -u origin <ветка> — отправляет новую ветку и устанавливает связь с origin (удалённым репозиторием).
  • Git fetch — загружает изменения с удалённого сервера без их применения к рабочей копии.
